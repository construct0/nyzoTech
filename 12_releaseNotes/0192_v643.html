Nyzo 643: bugfixes, gradle 8.4, subversion, git version tags, construct0 Apache 2.0 license

Nyzo version 643 (commits on construct0's GitHub) introduces a bugfix to the PreferencesUtil class, a bugfix to the Verifier class, an upgrade to gradle 8.4, the introduction of a subversion to the Version class and its references, introduction of git version tags for versions 642 and above, and a switch in license to the Apache 2.0 license within construct0's nyzoVerifier repository. 

    This version affects all run modes.
    
    This version spans across multiple commits, when in doubt use the git version tags as guidance.
    
    Version 643.2 concludes the final subversion of this version.

PreferencesUtil bugfix

The setup intructions pertaining to the preferences mention the usage of "true" and "false" for boolean values - this however did not work before (0 and 1 did), and usage of the string variants may have lead to misconfigured instances due to the missing checks.



Verifier connectivity, temporary threshold adjustment

It was observed that in-cycle nodes were incapable of being able to be properly connected with the rest of the cycle after a reboot or reload of the running nyzo_verifier instance.

This meant these nodes were unable to vote for or produce blocks while in this state.

This update temporarily sets the threshold to 51% + 1 node instead of the previous 75%. As is noted in the in-line documentation, this does not imply proper automatic consensus, but it allows for a properly connected and thus block (vote) producing verifier.

Thanks to EggPool for reporting this and suggesting a fix.





Gradle upgrade to version 8.4

 To make development using Visual Studio Code tooling possible, gradle was upgraded to version 8.4.

 This upgrade does not change the way you build the source code using existing scripts.


Introduction of subversion

Inspired by the Semantic Versioning methodology a subversion was introduced, allowing for versions to be released using a Major and a Minor version.

This version's final subversion is 2, making it v643.2.

All references where the Version class is used were updated as well.









Introduction of git repository version tags

To ease cloning or pulling a specific version to your machine, and in conjunction with the subversion addition, version tags were introduced to the construct0 nyzoVerifier repository for versions 642 and above.

Switch to Apache 2.0 License

Due to legal ambiguities surrounding the previous the Unlicense license and to provide ease of mind for developers within the European Union, the license was switched to the Apache 2.0 license.
