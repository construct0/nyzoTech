Game example

This page requires version 8 or later of the Chrome extension to demonstrate full functionality.

The game is fully playable in its current state, but this page and the Chrome extension are being updated to support pre-authorization of automatic transactions. Pre-authorization will provide protection against malicious automatic transactions. When these updates are complete, version 7 of the extension will be released on GitHub and submitted to the Chrome Web Store.

The previous example has been moved to the Micropay automatic transaction example page. This page will build upon that example to demonstrate gaming with Nyzo.

The API request for loading board state is sent and received, and transactions in the response are processed. The board then activates, allowing game play. Moves are now shown at a lower opacity until they are confirmed by the blockchain.

Updates made to the client in version 623 allow unconfirmed (pending) transactions to be retrieved in the transaction search. This enables faster game play. The practice of sending the current game state in transactions, not merely the latest move, eliminates the possibility that a bait-and-switch tactic would allow for successful game manipulation. Additionally, the use of a trusted client enables a high certainty that pending transactions will be confirmed.
